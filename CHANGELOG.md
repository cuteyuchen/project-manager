# 更新日志

## v1.5.1

### 新功能

- 新增多级项目工作区，根项目可关联扫描到的子项目，并在侧栏中选择子项目执行各自的脚本、自定义命令和 Git 操作。
- 批量导入与手动导入统一支持自动扫描目录中的子项目；Git 克隆完成后也会重新扫描并挂载识别到的子项目。
- 当根项目包含子项目且根目录本身是 Git 仓库时，子项目列表新增“根目录（Git）”虚拟入口，可直接查看根仓库状态并执行根级命令，不产生重复项目数据。
- uTools 与 ZTools 插件补齐子项目扫描和批量导入预扫描能力，与 Tauri 桌面端保持一致。
- 设置页新增全局自定义背景图片，支持本地图片选择、预览、清除和背景显示强度调节，桌面端与插件端均可使用。
- 父子项目钻取新增方向动画；返回父项目或主项目列表时恢复离开前的滚动位置。

### 优化

- 批量导入、手动导入和项目内扫描统一复用同一套子项目类型、脚本和包管理器转换逻辑。
- 优化子项目卡片布局，使收藏图标与项目名称保持视觉对齐。
- 自定义背景启用后，主工作区和侧栏自动使用半透明模糊层，兼顾背景展示与内容可读性。
- 优化项目列表、Node、端口、提交日历等页面在自定义背景下的透明层级，减少大面积不透明区域。
- 功能页切换由串行淡出淡入改为交叠式位移淡化，减少切换过程中的空白和停顿感。
- 所有弹窗统一增加背景高斯模糊与强化遮罩，减少自定义背景和列表内容透出造成的重影。
- 统一主要页面的玻璃模糊强度和最大内容宽度，并优化宽屏与小屏下的数据面板布局。
- 项目列表、Node、端口、提交日历和设置页统一使用相同的页面标题、操作区、间距与顶部玻璃层结构。
- 提交日历的跳过项目默认折叠显示，项目名称过长时省略，可按需展开全部内容。
- 父子项目的文件和备忘录改为分别保存与展示；无可运行命令或环境配置时隐藏对应功能入口。
- 包含子项目的项目列表始终显示同级“父项目”入口，可随时切回父项目自身的功能与数据。
- 所有导入入口在添加根项目及子项目后统一扫描前端环境变量和 Vite 代理配置。

### 修复

- 修复未初始化 Git 的子项目被误识别为父级 Git 仓库的问题；现在只有项目目录本身等于仓库根目录时才显示 Git 状态。

## v1.4.0

### 优化

- 新增全局主题变量与通用页面、面板、按钮、弹窗样式，统一主工作台、设置、Node、端口、Git、项目文件、备忘录等区域的视觉风格。
- 收敛组件内硬编码颜色和重复暗色分支，改为语义化 token 驱动，提升亮色 / 暗色模式的一致性。
- 统一 Git 工具栏、提交区、文件管理工具栏、右键菜单、预览弹窗、设置卡片与项目列表的圆角、阴影、间距和交互反馈。
- 补充全局 focus-visible、reduced-motion 与滚动条样式，文件管理组件模板 DOM ref 改用 `useTemplateRef`。
